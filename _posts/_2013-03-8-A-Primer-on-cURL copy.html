I was introduced to testing APIs long before I started web development. Perhaps the most indespensible resource available to me was cURL, which is a command line utility to send and receive data to servers. I've since encountered various alternatives, such as Python's PycURL, and Ruby's Typhoeus, however none of them has ever been as lightweight and easy to set up, use and understand as cURL. If you're on a Mac, or a Linux machine, it's as easy as opening up Terminal and entering some of the commands we'll go over below.

HTTP Requests

The REST API uses four main HTTP methods (although the list is expanding quickly) to interact with content: GET, PUT, POST & DELETE:

  • GET is used to retrieve content, without making any modifications
  • POST and PUT are used to create or update resources. There are subtle differences between them (generally a POST is used to update and element, or add to a queue, while a PUT is used to replace an element), but it depends somewhat on the program architecture. In this case, simply follow the REST API Wiki on which to use for each particular command since Madra makes use of both.
  • DELETE is used to remove a resource
